# 2026-03-28 10:40:29 人工验收清单

## 1. 验收目标

本轮人工验收用于确认当前原型已经从“接口原型”升级为“可打开、可看、可交互”的场景构建台雏形。

本轮重点验收：

- 首页是否存在
- 三栏结构是否清晰
- 是否可通过点击完成选择
- 是否可在页面上直接生成 `Scenario`
- 是否可在页面上直接生成 `Experience`
- 是否可在页面上直接记录 `Build Workflow`
- 错误是否可见

## 2. 启动前检查

在目录 `/home/wgw/CodexProject/NodeTest` 执行：

```bash
npm test
npm run lint
npm start
```

通过标准：

- `npm test` 全部通过
- `npm run lint` 无报错
- `npm start` 输出 `server listening on 3000`

## 3. 首页验收

浏览器打开：

- `http://127.0.0.1:3000/`

通过标准：

- 不再出现 `Cannot GET /`
- 页面标题为 `场景构建台`
- 页面可看到三栏：
  - `知识要素区`
  - `场景组合区`
  - `输出区`

## 4. 固定样本展示验收

在左栏确认：

- 能看到实体样本
- 能看到业务流程样本
- 能看到关系视角样本

通过标准：

- 左栏不是空白
- 样本项可勾选

## 5. 场景组合验收

步骤：

1. 勾选至少一个实体
2. 勾选至少一个业务流程
3. 可选勾选一个关系视角
4. 点击 `生成场景`

通过标准：

- 右栏自动切到或展示 `Scenario`
- 输出中包含：
  - `scenario_id`
  - `selected_entity_refs`
  - `selected_business_flow_refs`
  - `source_knowledge_refs`

## 6. 表达生成验收

步骤：

1. 先完成场景组合
2. 点击 `生成表达`

通过标准：

- 右栏能看到 `Experience`
- 输出中包含：
  - `experience_id`
  - `scenario_ref`
  - `experience_type`

## 7. 构建流程记录验收

步骤：

1. 先完成场景组合
2. 再完成表达生成
3. 点击 `记录构建流程`

通过标准：

- 右栏能看到 `Build Workflow`
- 输出中包含：
  - `build_workflow_id`
  - `target_scenario_ref`
  - `target_experience_refs`
  - `status`

## 8. 错误显示验收

步骤：

1. 刷新页面
2. 不做任何选择
3. 直接点击 `生成场景`

通过标准：

- 页面出现明确错误提示
- 错误内容可见，不是静默失败

然后继续测试：

1. 刷新页面
2. 不生成场景
3. 直接点击 `生成表达`

通过标准：

- 页面出现明确错误提示

## 9. 重置验收

步骤：

1. 完成若干勾选和生成动作
2. 点击 `重置`

通过标准：

- 选择状态被清空
- 输出状态回到初始态
- 错误提示消失

满足以下条件，可判定本轮通过人工验收：

- 可点击生成场景
