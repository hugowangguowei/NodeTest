# 2026-03-28 13:04:10 人工验收清单

## 1. 验收目标

本轮人工验收用于确认当前原型已经从“场景构建台雏形”升级为“软件工厂雏形”。

本轮重点验收：

- 首页术语是否切换为 `软件工厂`
- 三栏是否对应 `行业/领域知识仓库`、`软件工厂`、`定制软件定义区`
- 左栏是否同时展示知识仓库和状态集合
- 中栏是否可选择 `构建能力单元`
- 右栏是否优先展示 `软件定义`
- 主要链路是否仍可交互执行

## 2. 启动前检查

在目录 `/home/wgw/CodexProject/NodeTest` 执行：

```bash
npm test
npm run lint
npm start
```

通过标准：

- `npm test` 全部通过
- `npm run lint` 无报错
- `npm start` 输出 `server listening on 3000`

## 3. 首页语义验收

浏览器打开：

- `http://127.0.0.1:3000/`

通过标准：

- 页面标题为 `软件工厂`
- 页面主标题为 `软件工厂`
- 页面可看到三栏：
  - `行业/领域知识仓库`
  - `软件工厂`
  - `定制软件定义区`

## 4. 知识仓库展示验收

在左栏确认：

- 能看到 `仓库定位`
- 能看到 `状态集合`
- 能看到实体样本
- 能看到业务流程样本
- 能看到关系视角样本

通过标准：

- 左栏不是空白
- 状态集合为可见静态信息
- 实体、业务流程、关系视角可勾选

## 5. 软件工厂装配验收

在中栏确认：

- 能看到 `软件定义意图`
- 能看到 `构建能力单元`
- 能填写 `软件名称`、`使用期望`、`面向对象`

步骤：

1. 勾选至少一个实体
2. 勾选至少一个业务流程
3. 勾选至少一个构建能力单元
4. 可选勾选一个关系视角
5. 点击 `生成场景`

通过标准：

- 右栏展示 `软件定义`
- 输出中包含：
  - `software_intent`
  - `factory_assembly`
  - `generated_scenario`
  - `source_knowledge_refs`

## 6. 工作台表达验收

步骤：

1. 先完成软件定义生成
2. 点击 `生成表达`

通过标准：

- 右栏切到 `工作台表达`
- 输出中包含：
  - `software_terminal_view`
  - `experience_id`
  - `scenario_ref`
  - `state_sync_policy`

## 7. 构建流程验收

步骤：

1. 先完成软件定义生成
2. 再完成工作台表达生成
3. 点击 `记录构建流程`

通过标准：

- 右栏切到 `构建流程`
- 输出中包含：
  - `build_workflow_id`
  - `target_scenario_ref`
  - `target_experience_refs`
  - `operation_context`

## 8. 错误显示验收

步骤一：

1. 刷新页面
2. 不勾选任何实体、流程、能力
3. 直接点击 `生成场景`

通过标准：

- 页面出现明确错误提示

步骤二：

1. 刷新页面
2. 勾选实体和流程，但不勾选构建能力单元
3. 点击 `生成场景`

通过标准：

- 页面提示必须先选择构建能力单元

步骤三：

1. 刷新页面
2. 不生成软件定义
3. 直接点击 `生成表达`

通过标准：

- 页面出现明确错误提示

## 9. 重置验收

步骤：

1. 完成若干勾选和生成动作
2. 点击 `重置`

通过标准：

- 勾选状态被清空
- 右栏回到初始定义态
- 错误提示消失
