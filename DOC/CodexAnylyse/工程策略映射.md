# 工程策略映射

## 1. 本项目文档根

本项目 Codex 文档根固定为：

- `DOC/CodexAnylyse/`

## 2. 验收类工件目录

本项目以下工件统一放入：

- `DOC/CodexAnylyse/验收/`

适用类型：

- 人工验收清单
- 人工验收记录
- 验收结论记录

## 3. 验收类命名规则

验收类文件统一使用日期加时间命名：

- `YYYY-MM-DD-HHMMSS-人工验收清单.md`
- `YYYY-MM-DD-HHMMSS-人工验收记录.md`
- `YYYY-MM-DD-HHMMSS-验收结论记录.md`

这样可以保证同一天内生成多份验收文档时不冲突。

## 4. 非验收类工件

以下工件当前仍放在文档根：

- 分析文档
- 设计稿
- 契约草案
- 实现计划
- 自测记录
- 交接记录

## 5. 当前执行要求

后续在本项目内新增任何验收类文档时，必须同时满足：

1. 放入 `DOC/CodexAnylyse/验收/`
2. 文件名带日期和时间
3. 不得直接放在文档根
