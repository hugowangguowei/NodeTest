# P1.2 原型设计与交互方案

更新时间：2026-04-02
关联 Issue：#2

## 1. 页面结构

工作台采用三栏：

1. 知识要素区（可选标准知识、实体、行为、事件）
2. 场景组合区（组合流程并触发构建）
3. 输出结果区（展示场景/体验/流程定义）

## 2. 交互步骤

1. 页面加载时拉取工作台初始数据
2. 用户选择要素并提交组合请求
3. 页面展示组合结果并可继续生成体验定义
4. 页面展示流程定义用于后续实现

## 3. 交互与接口映射

| 交互步骤 | 接口 | 说明 |
|---|---|---|
| 页面初始化 | `GET /api/workbench/bootstrap` | 返回工作台基础数据 |
| 标准知识校验 | `POST /api/validate/standard-knowledge` | 输入合法性检查 |
| 场景组合 | `POST /api/scenarios/compose` | 生成场景结构 |
| 体验创建 | `POST /api/experiences/create` | 生成体验定义 |
| 流程生成 | `POST /api/build-workflows/create` | 生成流程定义 |

## 4. 验收脚本（最小）

1. 打开 `http://127.0.0.1:3000/`
2. 触发一次完整组合流程
3. 在页面看到结构化结果
4. 确认结果字段与接口返回一致
