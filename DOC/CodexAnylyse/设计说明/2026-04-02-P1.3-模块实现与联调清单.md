## 2. 模块职责

1. `contracts`：约束输入输出结构，防止无定义字段。
2. `domain`：承接场景/体验/流程组合逻辑。
3. `server`：暴露 HTTP 接口，连接前端交互。
4. `validation`：在进入领域逻辑前做数据校验。
5. `tests`：覆盖契约、领域、接口三个层次。

## 3. 联调检查点

1. `GET /api/workbench/bootstrap` 可返回基础数据。
2. `POST /api/scenarios/compose` 可返回场景结构。
3. `POST /api/experiences/create` 可返回体验定义。
4. `POST /api/build-workflows/create` 可返回流程定义。
5. `npm test` 全量通过。
