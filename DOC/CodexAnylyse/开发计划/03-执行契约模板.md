# 执行契约模板

更新时间：2026-04-02

## 1. 用途

当后续把工作切成可执行子任务时，执行契约应至少满足本模板。

该模板可用于：

- Issue
- Task
- 本地执行单
- 外部协作系统中的任务卡片

## 2. 模板

```md
目标：
- ...

Owner：
- ...

Write Scope：
- ...

Depends On：
- ...

验收入口：
- 主入口：...
- 辅助入口：...

验收标准：
- ...

输出物：
- ...

证据链接：
- 计划文档：...
- 设计文档：...
- 自测报告：...
- 验收清单：...
- 交接记录：...

当前状态：
- 待开发 / 开发中 / 已自测 / 待人工验收 / 已验收 / 阻塞中

上级节点：
- ...
```

## 3. 当前项目执行规则

在当前项目未接入外部 Tracker 前，如需新增细粒度任务，优先在本地采用该模板，再决定是否同步到外部工具。

## 4. 协作路径写法规则

用于 GitHub 协作（Issue / Project / PR 评论）时，路径统一按以下规则：

1. 优先使用仓库相对路径：
   - `DOC/CodexAnylyse/设计稿/00-平台总设计稿.md`
   - `src/server/routes.js`
2. 需要跨成员可点击时，补充仓库 URL：
   - `https://github.com/<owner>/<repo>/blob/<branch>/DOC/...`
3. 禁止写本机绝对路径：
   - `/home/wgw/...`
