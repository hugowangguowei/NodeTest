# 2026-04-02 P2.1 知识仓库对象模型契约草案

## 1. 文档定位

本稿是 P2.1 的对象模型契约草案，用于为后续 P2.2/P2.3 提供统一结构基线。

本稿只定义“对象与字段契约”，不定义 UI 或实现框架细节。

## 2. 顶层仓库契约

```json
{
  "repository_version": "2.1.0-draft",
  "repository_id": "repo-001",
  "domain_scope": "cross-domain",
  "state_domains": [],
  "objects": {
    "entities": [],
    "behaviors": [],
    "events": [],
    "relationships": [],
    "business_flows": [],
    "rules": []
  },
  "runtime_objects": {
    "flow_instances": [],
    "node_instances": []
  },
  "audit_objects": {
    "operation_logs": [],
    "state_snapshots": []
  }
}
```

## 3. 状态域对象契约

```json
{
  "id": "state-domain-001",
  "name": "知识语义域",
  "domain_type": "knowledge_semantic",
  "description": "承载实体、行为、事件、关系、流程、规则",
  "object_refs": [],
  "update_policy": "versioned_upsert"
}
```

字段约束：

1. `domain_type` 必须来自受控枚举：
   - `master_data`
   - `knowledge_semantic`
   - `process_runtime`
   - `projection_state`
   - `audit_trace`
2. `object_refs` 仅允许引用已注册对象。

```json
  "id": "flow-inst-001",
```

```json
  "completed_at": "2026-04-02T00:10:00Z",
```

### 5.3 `operation_log`

```json
{
  "id": "oplog-001",
  "operation_type": "state_update",
  "operator_ref": "entity-actor-001",
  "target_ref": "flow-inst-001",
  "before_snapshot_ref": "snapshot-001",
```

### 5.4 `state_snapshot`

```json
{
  "id": "snapshot-001",
  "target_ref": "flow-inst-001",
  "payload": {},
  "captured_at": "2026-04-02T00:09:59Z",
  "state_domain_ref": "state-domain-005"
}
```

## 6. 通用契约规则

1. 所有跨对象关联必须使用 `id`/`*_ref`/`*_refs`。
2. 所有对象必须带 `state_domain_ref`，禁止无域对象。
3. `audit_trace` 域对象只增不改，禁止覆盖历史日志。
4. 不允许使用页面术语（例如“左栏卡片”“tab 状态”）作为正式对象字段。
