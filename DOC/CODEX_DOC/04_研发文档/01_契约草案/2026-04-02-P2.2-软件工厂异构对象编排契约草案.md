# 2026-04-02 P2.2 软件工厂异构对象编排契约草案

## 1. 文档定位

本稿定义软件工厂异构对象编排的核心契约字段，作为 P2.2 的正式草案输入。

## 2. 异构对象描述契约

```json
{
  "object_id": "factory-object-001",
  "name": "流程规则技能包",
  "kind": "skill",
  "form": "package",
  "stage": "assembly",
  "lifecycle_status": "published",
  "io_contract_ref": "io-contract-001",
  "dependency_contract": {
    "required_refs": [],
    "optional_refs": []
  },
  "replaceability_level": "hot_swappable",
  "contract_version": "1.0.0"
}
```

字段约束：

1. `kind` 枚举：`skill/plugin/template/component/rule_pack/connector`
2. `form` 枚举：`schema/config/script/package/remote_service`
3. `stage` 枚举：`definition/assembly/runtime/governance`
4. `lifecycle_status` 枚举：`draft/verified/published/deprecated/retired`

## 3. 编排请求契约

```json
{
  "request_id": "assembly-request-001",

  "selected_object_refs": [
    "factory-object-001",
    "factory-object-002"

  "constraints": {
    "require_published": true,
    "allow_deprecated": false

```

## 6. 接口契约草案

### 6.1 Catalog.Query

输入：

```json
{
  "kind": "skill",
  "stage": "assembly",
  "lifecycle_status": "published"
}
```

输出：

```json
{
  "items": [],
  "total": 0
}
```

### 6.2 Compatibility.Check

输入：

```json
{
  "object_refs": [],
  "constraints": {}
}
```

输出：

```json
{
  "compatible": true,
  "conflicts": []
}
```

### 6.3 Assembly.Plan / Assembly.Execute

输入：见第 3、4 章  
输出：见第 4、5 章

## 7. 错误码草案

| code | 含义 |
|---|---|
| `factory_object_not_found` | 编排对象不存在 |
| `factory_object_not_published` | 对象未发布不可编排 |
| `factory_dependency_conflict` | 依赖冲突 |
| `factory_io_contract_mismatch` | 输入输出契约不匹配 |
| `factory_plan_cycle_detected` | 编排图出现环依赖 |
| `factory_execution_step_failed` | 执行步骤失败 |

## 8. 与后续节点关系

1. 输入给 P2.3：统一回写时，将 `execution_report.operation_log_refs` 作为回写链路输入证据。
2. 输入给 P3：定制软件定义态可直接引用编排计划结果构建终端视图。
