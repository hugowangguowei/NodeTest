# 2026-04-03 P2.1 知识仓库对象模型标准规范

## 1. 文档定位

本稿是 P2.1 的对象模型标准规范，用于为后续 P2.2/P2.3/P2.4 提供统一结构基线。

本稿先定义“知识仓库本体（聚合根）”，再定义状态域与对象字段规范。状态域属于仓库组成部分，不替代仓库本体。

## 2. 顶层仓库聚合根规范

```json
{
  "knowledge_repository": {
    "repository_id": "repo-001",
    "repository_version": "2.1.0-draft",
    "meta": {
      "name": "跨领域知识仓库",
      "owner": "platform-team",
      "taxonomy_version": "v1"
    },
    "scope": {
      "domain_scope": "cross-domain",
      "tenant_mode": "single-tenant"
    },
    "domain_catalog": {
      "state_domains": []
    },
    "object_catalog": {
      "semantic_objects": {
        "entities": [],
        "behaviors": [],
        "events": [],
        "relationships": [],
        "business_flows": [],
        "rules": []
      },
      "runtime_objects": {
        "flow_instances": [],
        "node_instances": []
      },
      "audit_objects": {
        "operation_logs": [],
        "state_snapshots": []
      }
    },
    "indexes": {
      "id_index": {},
      "type_index": {},
      "domain_index": {}
    },
    "writeback_policy": {
      "mode": "append_with_validation",
      "conflict_strategy": "reject_and_report"
    },
    "audit_policy": {
      "retention": "long_term",
      "mutable": false
    }
  }
}
```

字段约束：

1. `knowledge_repository` 是唯一聚合根，所有对象必须可追溯到该根。
2. `domain_catalog.state_domains` 是状态域注册清单，业务对象只能引用已注册域。
3. `object_catalog` 用于对象托管与生命周期管理，不允许对象游离于目录外。
4. `indexes` 是仓库级索引，不等同于业务对象字段。

## 3. 状态域对象规范（仓库组成）

```json
{
  "id": "state-domain-001",
  "repository_ref": "repo-001",
  "name": "知识语义域",
  "domain_type": "knowledge_semantic",
  "description": "承载实体、行为、事件、关系、流程、规则",
  "object_refs": [],
  "update_policy": "versioned_upsert"
}
```

字段约束：

1. `domain_type` 必须来自受控枚举：
   - `master_data`
   - `knowledge_semantic`
   - `process_runtime`
   - `projection_state`
   - `audit_trace`
2. `repository_ref` 必须指向已存在仓库聚合根。
3. `object_refs` 仅允许引用 `object_catalog` 已注册对象。

## 6. 通用规范规则

1. 所有对象必须归属 `knowledge_repository`，并具备 `repository_ref`。
2. 所有跨对象关联必须使用 `id`/`*_ref`/`*_refs`。
3. 所有业务对象必须带 `state_domain_ref`，禁止无域对象。
4. `audit_trace` 域对象只增不改，禁止覆盖历史日志。
5. 不允许使用页面术语（例如“左栏卡片”“tab 状态”）作为正式对象字段。
