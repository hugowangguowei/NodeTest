# 2026-04-03 P2.1.1 知识仓库可视化映射标准规范

## 1. 文档定位

本规范定义知识仓库对象模型到可视化图谱的标准映射规则，约束节点类型、边类型、字段映射和筛选行为。

## 2. 输入与输出

输入：

- `knowledge_repository` 样例对象（P2.1 标准规范目录）

输出：

- 图谱结构：
  - `nodes[]`
  - `edges[]`
  - `stats`

## 3. 节点映射规则

| 源对象 | 节点 `node_type` | 主键 |
|---|---|---|
| `knowledge_repository` | `knowledge_repository` | `repository_id` |
| `state_domains[]` | `state_domain` | `id` |
| `semantic_objects.entities[]` | `entity` | `id` |
| `semantic_objects.behaviors[]` | `behavior` | `id` |
| `semantic_objects.events[]` | `event` | `id` |
| `semantic_objects.relationships[]` | `relationship` | `id` |
| `semantic_objects.business_flows[]` | `business_flow` | `id` |
| `semantic_objects.rules[]` | `rule` | `id` |
| `runtime_objects.flow_instances[]` | `flow_instance` | `id` |
| `runtime_objects.node_instances[]` | `node_instance` | `id` |
| `audit_objects.operation_logs[]` | `operation_log` | `id` |
| `audit_objects.state_snapshots[]` | `state_snapshot` | `id` |
| 未注册引用（兜底） | `reference` | 引用值 |

补充说明：

1. 角色实体（如董事长、总经理）在图谱中仍按 `entity` 节点类型表示，`raw.entity_type=person_role`。
2. `business_flow.node_refs` 若未在对象目录定义，会生成 `flow_node` 占位节点。

## 4. 边映射规则

| 关系语义 | `edge_type` | 方向 |
|---|---|---|
| 仓库包含状态域 | `contains_domain` | `repository -> state_domain` |
| 仓库包含对象 | `contains_object` | `repository -> object` |
| 状态域包含对象 | `contains` | `state_domain -> object` |
| 行为执行主体 | `acts` | `actor -> behavior` |
| 行为作用目标 | `targets` | `behavior -> target` |
| 行为受规则约束 | `constrained_by` | `behavior -> rule` |
| 行为触发事件 | `triggers` | `behavior -> event` |
| 关系对象语义边 | `relationship_type` 原值 | `from_ref -> to_ref` |
| 流程包含节点 | `has_node` | `flow -> flow_node` |
| 角色参与流程 | `participates_in` | `role -> flow` |
| 规则作用范围 | `applies_to` | `rule -> scope_ref` |
| 节点实例归属流程实例 | `instance_of_flow` | `node_instance -> flow_instance` |
| 节点实例对应定义节点 | `instance_of_node` | `node_instance -> flow_node` |
| 流程实例对应流程定义 | `instance_of` | `flow_instance -> business_flow` |
| 流程实例当前节点 | `current_node` | `flow_instance -> flow_node` |
| 审计操作主体 | `operates` | `operator -> operation_log` |
| 审计操作目标 | `targets` | `operation_log -> target` |
| 快照对应对象 | `snapshot_of` | `state_snapshot -> target` |

## 5. 查询与筛选规则

接口：`GET /api/visualization/knowledge-graph`

参数：

1. `include_audit`：`true/false`，默认 `true`。
2. `state_domain_ref`：按状态域过滤节点和边。
3. `node_type`：按节点类型过滤。

过滤原则：

1. 过滤后边必须满足“源节点和目标节点都在结果集内”。
2. 过滤结果保留仓库根节点以维持图谱上下文。

## 6. 一致性约束

1. 任一对象节点若有 `state_domain_ref`，其值必须存在于状态域节点集合。
2. 任一边的 `source/target` 必须对应现存节点，不存在时生成 `reference` 节点兜底。
3. 映射层不得修改源对象语义字段，仅允许补齐可视化辅助字段（如节点类别）。

## 7. 验收要点

1. 角色链路可见：执行人员 -> 合规主管 -> 财务主管 -> 总经理。
2. 流程链路可见：`business_flow` 与 `flow_node` 连接完整。
3. 审计开关可用：`include_audit=false` 时不返回审计对象节点。
