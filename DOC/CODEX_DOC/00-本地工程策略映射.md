## 4. 计划与契约规则

1. `03_研制计划/` 中的稳定 WBS 节点文档使用 `NN-WBS-节点编号-主题-研制计划.md`。
2. 当前已规范化的主计划文件：
   - `01-WBS-L0-NodeTest-研发总纲-研制计划.md`
   - `02-WBS-L1-P1-平台定义与原型收敛-研制计划.md`
   - `03-WBS-L2-P1-子节点清单文档-研制计划.md`
3. 执行契约索引放在 `03_研制计划/`，执行契约实例放在 `03_研制计划/02_执行契约/`。

## 5. 测试与验收工件规则

测试与验收工件只允许落在 `05_测试文档/` 对应子目录：

1. 自测报告 -> `01_自测报告/`
2. 验收清单 -> `02_验收清单/`
3. 验收记录 -> `03_验收记录/`
4. 验收结论 -> `04_验收结论/`

命名统一使用 `YYYY-MM-DD-HHMMSS-主题*.md`。  
本轮已将历史中缺少时分秒的两份自测文档补齐为时间戳格式，避免同日冲突。

## 6. 协作路径规则

在 GitHub Issue / Project / PR 评论中：

1. 默认使用仓库相对路径，例如 `DOC/CODEX_DOC/03_研制计划/...`、`src/server/routes.js`
2. 需要可点击时补充仓库 URL
3. 禁止使用本机绝对路径（如 `/home/wgw/...`）作为协作路径
