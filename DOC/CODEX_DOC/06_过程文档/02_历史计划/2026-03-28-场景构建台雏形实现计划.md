# Scenario Builder Prototype Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a first interactive `场景构建台` page so a human can select fixed sample knowledge, generate `Scenario`, generate `Experience`, and record `Build Workflow` from the browser.

**Architecture:** Add a minimal server-rendered frontend slice to the existing Express app. Keep the data source fixed and local, expose a small bootstrap API for the page, and continue to use the existing backend composition endpoints for all generation actions.

**Tech Stack:** Node.js CommonJS, Express, vanilla HTML/CSS/JS, `node:test`, Supertest, ESLint
