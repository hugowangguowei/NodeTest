# 文档目录与命名约定

更新时间：2026-03-29

## 1. 文档目录总规则

从现在开始，`DOC/CODEX_DOC/` 采用“根目录少量长期入口 + 分类目录承载阶段工件”的管理方式。

### 1.1 根目录只保留

- 工程总体分析
- 阅读顺序或索引
- 当前工程工作策略
- 文档目录与命名约定

### 1.2 分类目录承载

- `设计稿/`
- `设计说明/`
- `契约草案/`
- `开发计划/`
- `开发计划/历史计划/`
- `计划管理归档/`
- `自测报告/`
- `会话交接/`
- `验收清单/`
- `验收记录/`
- `验收结论/`

## 2. 命名规则

### 2.1 长期入口文档

长期入口文档推荐使用固定序号前缀，例如：

- `00-工程总体分析.md`
- `01-设计与计划阅读顺序.md`
- `02-当前工程Codex工作策略.md`

### 2.2 WBS 稳定计划文档

稳定计划文档按 WBS 节点命名，不使用“当前阶段计划”作为稳定文件名。

推荐格式：

- `01-WBS-L0-<project>-研发总纲.md`
- `02-WBS-L1-<phase>.md`

当前活动节点通过：

- `开发计划/00-开发计划README.md`

标识，而不是通过频繁改计划文件名表达。

### 2.3 阶段性文档

阶段性文档统一带日期，必要时带时间。

推荐格式：

- `YYYY-MM-DD-主题.md`
- `YYYY-MM-DD-HHMM-主题.md`
- `YYYY-MM-DD-HHMMSS-主题.md`

### 2.4 验收类文档

验收类文档必须带日期和时间，格式固定为：

- `YYYY-MM-DD-HHMMSS-人工验收清单.md`
- `YYYY-MM-DD-HHMMSS-人工验收记录.md`
- `YYYY-MM-DD-HHMMSS-验收结论记录.md`

## 3. 验收类文档放置规则

不同验收工件必须进入不同目录：

- 验收清单进入 `验收清单/`
- 验收记录进入 `验收记录/`
- 验收结论进入 `验收结论/`

不得再将三类文件混放到同一个 `验收/` 目录。

## 4. 当前执行规则

新增文档时，优先判断它属于哪一类，再决定目录，而不是先生成文件再随手堆到根目录。

计划管理过程说明、已失效的计划面讨论和不再属于活动计划面的协调笔记，应进入：

- `计划管理归档/`

## 5. 当前结论

本项目文档目录已经完成从“平铺堆积”向“分类管理”的迁移，后续新增文档必须沿用本约定。
