# P2.3 仓库统一更新链路与验证方案

更新时间：2026-04-02
关联 Issue：#11

## 1. 文档目标

本稿定义“定制软件对仓库操作统一回写”的最小可运行链路与验证方案，确保回写语义可验证、关键链路可观察。

## 2. 实现范围

本轮实现范围：

1. 回写请求契约（`writebackRequest`）。
2. 回写存储与操作日志（内存态最小实现）。
3. 回写接口与可观察查询接口。
4. 前端触发入口（执行统一回写按钮与结果 Tab）。
5. 集成测试与合同测试覆盖。

非范围：

1. 持久化数据库。
2. 分布式事务与重试队列。
3. 多租户隔离与权限治理。

## 3. 统一回写链路

链路步骤：

1. 用户完成场景、表达、构建流程与工厂编排后点击“执行统一回写”。
2. 前端构造 `writebackRequest` 并调用 `POST /api/repository/writeback`。
3. 服务端执行契约校验（`validators.writebackRequest`）。
4. 回写存储模块按 `target_domain + target_ref + mutation_type` 应用变更。
5. 系统生成操作日志（`operation_log_id`）并返回回写结果。
6. 前端拉取 `GET /api/repository/writeback/logs` 展示可观察输出。

## 4. 回写语义规则

### 4.1 目标域规则

`target_domain` 仅允许：

1. `master_data`
2. `knowledge_semantic`
3. `process_runtime`
4. `projection_state`
5. `audit_trace`

### 4.2 变更规则

`mutation_type` 语义：

1. `patch`：对象浅合并更新。
2. `replace`：目标对象整体替换。
3. `append`：数组追加写入。

### 4.3 追溯规则

每次回写必须携带：

1. `operation_id`
2. `trace.scenario_ref`
3. `trace.experience_ref`
4. `trace.workflow_ref`
5. `requested_at`

## 5. 可观察输出

### 5.1 接口

1. `POST /api/repository/writeback`
   - 回写执行并返回 `operation_log_id` 与 `next_snapshot`。
2. `GET /api/repository/writeback/logs`
   - 返回最近 N 条回写日志。
3. `GET /api/repository/writeback/snapshot`
   - 返回当前各状态域快照与日志计数。

### 5.2 页面

1. 中栏新增动作：`执行工厂编排`、`执行统一回写`。
2. 右栏新增 Tab：`回写结果`。
3. 回写结果展示：
   - 回写请求体
   - 回写响应结果
   - 最近日志列表

## 6. 关键验证脚本

### 6.1 自动化验证

```bash
npm test
```

覆盖点：

1. `tests/contracts/contractValidation.test.js`
   - `writebackRequest` 契约校验。
2. `tests/integration/httpApi.test.js`
   - 回写成功链路与可观察接口。
   - 回写非法请求返回 `400`。

### 6.2 手工链路验证

1. 打开 `http://127.0.0.1:3000/`。
2. 依次点击：
   - `生成场景`
   - `生成表达`
   - `记录构建流程`
   - `执行工厂编排`
   - `执行统一回写`
3. 切换到“回写结果”Tab，确认存在：
   - `operation_log_id`
   - `next_snapshot`
   - `latest_logs`

## 7. 需求正向追溯（P2.2 -> P2.3）

| 来源 | P2.3 响应 |
|---|---|
| P2.2 编排执行报告 `operation_log_refs` | 回写接口返回并记录 `operation_log_id` |
| P2.2 接口边界（Execute -> Report） | `writeback` + `logs/snapshot` 三接口 |
| Issue #11 验收：语义可验证 | 契约校验 + 集成测试 |
| Issue #11 验收：链路可观察 | logs/snapshot 接口 + 前端回写结果 Tab |

## 8. 需求逆向追溯（实现项 -> 验收）

| 实现项 | 对应验收标准 | 验证方式 |
|---|---|---|
| `writebackRequest` schema | 统一回写语义可验证 | 合同测试 |
| `unifiedWritebackStore` | 回写链路具备可执行语义 | 集成测试 |
| `POST /api/repository/writeback` | 关键回写链路可用 | 集成测试 |
| `GET /api/repository/writeback/logs` | 可观察输出 | 集成测试/页面 |
| 页面“执行工厂编排 -> 执行统一回写” | 主入口可人工验收 | 浏览器手工验收 |
