# P1.2 原型设计与交互方案

更新时间：2026-04-02
关联 Issue：#2

## 1. 文档目标与输入边界

本稿用于把 P1.1 的需求定义落到可执行的浏览器原型交互方案，覆盖页面结构、交互链路、接口映射和需求追溯。

输入依据：

1. `DOC/CODEX_DOC/02_设计说明/2026-04-02-P1.1-需求分析.md`
2. `public/index.html`
3. `public/app.js`
4. `src/server/routes.js`

输出边界：

1. 本稿只定义 P1 原型层交互，不扩展生产级 UI/权限/部署能力。
2. 本稿关注“定义态可演示闭环”，对应链路：
   `选知识 -> 组场景 -> 配表达 -> 记录构建流程`。

## 2. 页面结构（三区明细）

### 2.1 知识仓库区（左栏）

目标：让用户可视化选择“知识对象 + 状态集合 + 关系视角”，作为场景构建输入。

包含内容：

1. 仓库定位卡片
   - 展示 `repositoryMeta`：仓库名称、领域范围、定位、统一回写特征。
2. 状态集合卡片
   - 展示 `stateCollections`：基础资料状态集、事件流程状态集、更新策略。
3. 实体选择卡片
   - 来自 `knowledge.entities`，支持多选。
4. 业务流程选择卡片
   - 来自 `knowledge.business_flows`，支持多选。
5. 关系视角选择卡片
   - 来自 `relationshipViews`，支持多选。

交互约束：

1. 至少选择 1 个实体和 1 个业务流程，才允许生成场景。
2. 选择变化实时同步“当前工厂摘要”和输出预览。

### 2.2 软件工厂区（中栏）

目标：承接“软件定义态”输入和装配动作，形成可计算的场景/表达/流程定义。

包含内容：

1. 软件定义意图表单
   - `software_definition_id`
   - `software_name`
   - `usage_expectation`
   - `software_description`
   - `target_users`
2. 构建能力单元
   - 来自 `factoryCapabilities`，支持多选。
   - 能力类型覆盖 `skill/component/plugin`。
3. 当前工厂摘要
   - 实时显示：已选实体、流程、关系视角、能力数量。
4. 操作按钮
   - `生成场景`
   - `生成表达`
   - `记录构建流程`
   - `重置`

交互约束：

1. 未选能力单元时禁止生成场景。
2. 未生成场景时禁止生成表达。
3. 未生成场景与表达时禁止记录构建流程。

### 2.3 输出结果区（右栏）

目标：将软件定义态结果结构化展示，供人工验收和后续模块实现追溯。

包含内容：

1. 软件定义 Tab
   - 展示 `software_intent`、`selected_knowledge`、`factory_assembly`、`generated_scenario`。
2. 工作台表达 Tab
   - 展示 `software_terminal_view` 与状态回写策略。
3. 构建流程 Tab
   - 展示 `build_workflow` 与操作上下文。
4. 错误信息 Tab + Banner
   - 显示交互校验错误和接口返回错误。

## 3. 关键交互流程

### 3.1 主流程

1. 初始化
   - `GET /api/workbench/bootstrap` 拉取固定样本和模板，填充三区基础数据。
2. 选择知识与装配能力
   - 用户勾选实体/业务流程/关系视角/能力单元，填写软件定义意图。
3. 生成场景
   - `POST /api/scenarios/compose` 返回 `scenario`。
4. 生成表达
   - `POST /api/experiences/create` 返回 `experience`。
5. 记录构建流程
   - `POST /api/build-workflows/create` 返回 `workflow`。
6. 输出复核
   - 在右栏切换 Tab 校验三类输出结构一致性。

### 3.2 异常分支

1. 条件不满足
   - 前端即时报错（例如“至少选择一个实体和一个业务流程”）。
2. 接口失败
   - 错误写入 `error banner` 与错误 Tab，保留当前成功步骤结果，便于定位问题。

## 4. 交互与接口映射

| 交互点 | 接口 | 请求关键字段 | 响应关键字段 | 前端落点 |
|---|---|---|---|---|
| 页面初始化 | `GET /api/workbench/bootstrap` | 无 | `repositoryMeta` `knowledge` `stateCollections` `factoryCapabilities` `scenarioDraft` `experienceTemplate` | 左/中栏基础数据 |
| 标准知识校验（预留） | `POST /api/validate/standard-knowledge` | `knowledge` | `status=valid` 或 `errors` | 错误 Banner |
| 生成场景 | `POST /api/scenarios/compose` | `knowledge` + `selection`（实体/流程/关系/能力） | `scenario_id` `selected_*` `relationship_views` `capabilities` | 右栏“软件定义” |
| 生成表达 | `POST /api/experiences/create` | `scenario` + `selection`（role/views/cards/actions） | `experience_id` `workspace_cards` `quick_actions` `flow_projections` | 右栏“工作台表达” |
| 记录构建流程 | `POST /api/build-workflows/create` | `scenario` + `experiences` + `operationLogs` | `build_workflow_id` `steps` `operation_log_refs` | 右栏“构建流程” |

## 5. 需求正向追溯（P1.1 -> P1.2）

| P1.1 需求锚点 | P1.2 设计实现 |
|---|---|
| 3.1.2 选要素-组场景-配表达-生成定义 | 第 3 章主流程与第 4 章接口链路 |
| 4.3 三大模块边界（仓库/工厂/终端） | 第 2 章三区结构与职责划分 |
| 5.2 全阶段需求中的 P1 定义态闭环 | 第 3.1 节步骤 1~6 |
| 5.3 跨阶段通用能力（对象关系、回写一致性） | 第 2.1/2.3 节的状态集合与状态回写策略展示 |
| 6.1 可理解性、6.2 可追溯性 | 第 2.3 节结构化输出 + 第 4 章接口字段映射 |

## 6. 需求逆向追溯（设计项 -> 需求）

| 原型设计项 | 对应 P1.1 需求 | 验收方式 |
|---|---|---|
| 左栏知识仓库选择 | 4.3-知识仓库边界、5.2-P1 定义态闭环 | 勾选实体/流程可驱动场景生成 |
| 中栏软件工厂装配 | 4.3-软件工厂边界、3.1.2 标准化链路 | 通过按钮链路完成 3 次 POST |
| 右栏输出结构 | 4.3-自定义终端边界、6.2 可追溯性 | 三个 Tab 均展示结构化 JSON |
| 错误反馈机制 | 6.1 可理解性、6.3 可验证性 | 触发条件不满足时出现可读错误 |

## 7. 最小验收脚本

1. 启动服务后打开 `http://127.0.0.1:3000/`。
2. 在左栏选择至少 1 个实体、1 个业务流程、1 个关系视角。
3. 在中栏填写软件定义名称并选择至少 1 个能力单元。
4. 依次点击 `生成场景 -> 生成表达 -> 记录构建流程`。
5. 在右栏 `软件定义/工作台表达/构建流程` 三个 Tab 中确认都有结构化结果。
6. 点击 `重置`，确认选择状态与输出恢复初始状态。
