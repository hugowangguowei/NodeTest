# 人工验收清单 - P2.3统一回写链路与验证

时间：2026-04-02 22:46:23

## 1. 验收目标

确认统一回写链路可执行、可观察、可验证。

## 2. 验收步骤

1. 启动服务后打开 `http://127.0.0.1:3000/`
2. 在页面依次执行：生成场景 -> 生成表达 -> 记录构建流程 -> 执行统一回写
3. 打开“回写结果”Tab，确认有：
   - `operation_log_id`
   - `next_snapshot`
   - `latest_logs`
4. 直接访问 `GET /api/repository/writeback/logs?limit=5`，确认返回日志数组
5. 直接访问 `GET /api/repository/writeback/snapshot`，确认返回域快照和日志计数
6. 执行 `npm test`，确认测试通过

## 3. 通过标准

- [ ] 统一回写语义可验证（契约校验有效）
- [ ] 关键回写链路可执行（POST writeback 成功）
- [ ] 回写链路具备可观察输出（logs/snapshot 与页面 Tab）
- [ ] 自动化测试通过
