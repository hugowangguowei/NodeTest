## 5. 场景组合验收

执行：

```bash
jq -n \
  --slurpfile knowledge tests/fixtures/standardKnowledge.valid.json \
  --slurpfile selection tests/fixtures/scenario.selection.json \
  '{knowledge: $knowledge[0], selection: $selection[0]}' \
| curl -X POST http://127.0.0.1:3000/api/scenarios/compose \
  -H 'Content-Type: application/json' \
  --data @-
```

通过标准：

- 返回对象中包含：

  - `selected_entity_refs` 含 `entity-001`
  - `selected_business_flow_refs` 含 `business-flow-001`
  - `source_knowledge_refs` 含 `sk-001`

## 6. 表达投影验收

说明：

- 当前 `experience.selection.json` 依赖 `relationship-view-001`
- 因此验收时不能直接拿默认场景组合结果原样调用
- 需手动构造一个包含 `relationship_views` 的最小场景输入

执行：

```bash
jq -n \
  --slurpfile selection tests/fixtures/experience.selection.json \
  '{
    scenario: {
      scenario_id: "scenario-001",
      selected_business_flow_refs: ["business-flow-001"],
      relationship_views: [{id: "relationship-view-001"}]
    },
    selection: $selection[0]
  }' \
| curl -X POST http://127.0.0.1:3000/api/experiences/create \
  -H 'Content-Type: application/json' \
  --data @-
```

通过标准：

- 返回对象中包含：
  - `experience_id = "experience-001"`
  - `scenario_ref = "scenario-001"`
  - `experience_type = "personal_workspace"`
  - `flow_projections[0].business_flow_ref = "business-flow-001"`


### 8.2 表达超出场景作用域应失败

执行：

```bash
cat <<'JSON' | curl -X POST http://127.0.0.1:3000/api/experiences/create \
  -H 'Content-Type: application/json' \
  --data @-
{
  "scenario": {
    "scenario_id": "scenario-001",
    "selected_business_flow_refs": [],
    "relationship_views": []
  },
  "selection": {
    "experience_id": "experience-001",
    "experience_type": "personal_workspace",
    "name": "事项工作台",
    "description": "越权测试",
    "role_views": [],
    "workspace_cards": [],
    "quick_actions": [],
    "flow_projections": [
      {
        "id": "flow-projection-001",
        "name": "事项流程透视",
        "business_flow_ref": "business-flow-001",
        "focus_node_refs": [],
        "display_preferences": {}
      }
    ],
    "relationship_projections": [],
    "layout_preferences": {},
    "interaction_preferences": {}
  }
}
JSON
```

通过标准：

- 当前服务应返回错误
- 表示表达层不能越权引用场景外流程

说明：

- 当前版本还没有统一错误响应格式
- 这里只验“失败”这一行为，不验错误结构美观度
